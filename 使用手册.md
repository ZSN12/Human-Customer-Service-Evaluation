# 人工客服质检系统使用手册

## 1. 系统简介

人工客服质检系统是一款基于规则引擎和大语言模型（LLM）的混合质检工具，旨在帮助企业提升客服质量，规范服务标准，提高客户满意度。

### 1.1 核心功能

- **批量质检**：支持上传大量会话记录进行批量分析
- **智能分析**：结合规则引擎和LLM进行深度分析
- **可视化报告**：提供多维度数据可视化和详细分析报告
- **实时监控**：实时展示质检进度和费用统计
- **自定义规则**：支持自定义评分标准和质检规则
- **历史趋势**：跟踪客服质量的历史变化趋势

### 1.2 技术架构

- **前端**：Streamlit框架，提供直观的Web界面
- **后端**：Python，处理数据和业务逻辑
- **AI服务**：集成DeepSeek大语言模型
- **数据存储**：本地数据库
- **数据可视化**：Plotly库

## 2. 安装与配置

### 2.1 环境要求

- Python 3.8+
- 稳定的网络连接（用于调用LLM API）
- 足够的磁盘空间（存储会话数据和报告）

### 2.2 安装步骤

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd <项目目录>
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **配置环境变量**
   - 复制`.env.example`文件为`.env`
   - 在`.env`文件中填写DeepSeek API密钥
   ```
   # LLM Configuration
   DEEPSEEK_API_KEY=your_deepseek_api_key_here
   DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
   DEEPSEEK_MODEL=deepseek-chat
   
   # LLM Settings
   LLM_MAX_RETRIES=2
   LLM_TIMEOUT=30
   ENABLE_LLM_VERIFICATION=true
   ```

### 2.3 启动系统

```bash
python app.py
```

系统会自动打开浏览器，显示登录界面。

## 3. 使用流程

### 3.1 数据上传

1. **准备数据文件**：确保文件包含以下列：
   - 会话ID：唯一标识每个会话
   - Message：消息内容
   - Type：消息类型（5表示人工回复）

2. **上传文件**：
   - 在左侧边栏的"数据上传"区域点击"上传会话记录文件"
   - 选择支持的文件格式（.xlsx、.xls、.csv）
   - 系统会自动分析文件并显示会话数量

3. **字段映射**：如果文件列名与要求不符，系统会提示进行字段映射

### 3.2 开始质检

1. **确认上传成功**：系统会显示文件上传成功信息，包括行数和会话数

2. **点击"开始质检"**：系统会开始批量分析会话

3. **查看进度**：系统会显示实时进度和预计剩余时间

4. **查看费用**：系统会实时显示累计费用和Token使用情况

### 3.3 查看结果

1. **核心指标**：查看总会话数、平均得分、合格率等核心指标

2. **可视化分析**：
   - 质检维度分析（雷达图）
   - 得分分布（饼图）
   - 高频错误维度（柱状图）
   - 高频错误类型（饼图）

3. **质检结果明细**：
   - 查看详细的质检结果
   - 按得分或会话ID排序
   - 分页查看结果

4. **查看证据**：选择会话ID查看详细的质检证据

5. **人工复核**：修改得分并保存

### 3.4 下载报告

1. **生成报告**：质检完成后，系统会自动生成Excel格式的报告

2. **下载报告**：在左侧边栏的"报告下载"区域点击"下载质检报告"

3. **报告内容**：
   - 质检明细：详细的会话质检结果
   - 会话汇总：会话级别的汇总信息
   - 高频扣分项：常见错误的统计
   - 维度分析：各维度的表现分析
   - 得分分布：得分区间的分布情况

## 4. 功能详解

### 4.1 核心指标

- **总会话数**：上传文件中的会话总数
- **平均得分**：所有会话的平均质检得分
- **合格率**：得分≥60分的会话占比
- **实际费用**：调用LLM API产生的实际费用

### 4.2 可视化分析

#### 4.2.1 质检维度分析
- **雷达图**：展示各维度的平均得分
- **维度包括**：对话逻辑合规性、商品/订单信息准确性、平台政策合规性、服务质量、销售转化能力

#### 4.2.2 得分分布
- **饼图**：展示不同得分区间的会话占比
- **得分区间**：0-60、61-70、71-80、81-90、91-100

#### 4.2.3 高频错误分析
- **高频错误维度**：展示错误出现最多的前5个维度
- **高频错误类型**：展示不同错误类型的分布情况

### 4.3 质检结果明细

- **会话ID**：会话的唯一标识
- **最终得分**：质检后的最终得分
- **扣分项编码**：命中的扣分项编码
- **优化建议**：针对会话的具体优化建议
- **筛选功能**：按最低得分筛选结果
- **排序功能**：按最终得分或会话ID排序
- **分页功能**：支持每页显示20、50、100条记录

### 4.4 历史趋势分析

- **均分趋势**：每日平均得分的变化趋势
- **合格率趋势**：每日合格率的变化趋势
- **费用趋势**：每日质检费用的变化趋势
- **会话数量**：每日处理的会话数量

### 4.5 系统设置

#### 4.5.1 评分标准管理
- **初始满分**：设置初始满分（默认100分）
- **最低分限制**：设置最低分（默认0分）
- **维度配置**：修改各维度的基础扣分
- **扣分项配置**：调整扣分项的严重程度

#### 4.5.2 规则管理
- **查看规则**：查看当前启用的规则
- **启用/禁用规则**：根据需要启用或禁用规则
- **编辑规则**：修改现有规则的参数
- **删除规则**：删除不需要的规则
- **添加新规则**：创建新的质检规则
  - 规则类型：关键词、正则表达式、重复检测、模板匹配、解决方案匹配

#### 4.5.3 数据库与缓存
- **清除缓存**：清除系统缓存数据
- **数据库管理**：管理系统数据库

## 5. 常见问题与解决方案

### 5.1 数据上传问题

**问题**：上传文件后系统提示字段映射
**解决方案**：根据系统提示，选择文件中的列对应"会话ID"、"Message"、"Type"字段

**问题**：文件过大无法上传
**解决方案**：系统限制最大行数为20万行，超过限制的文件需要拆分后上传

### 5.2 质检问题

**问题**：质检过程中出现错误
**解决方案**：查看日志信息，检查API密钥是否正确，网络连接是否稳定

**问题**：质检结果不准确
**解决方案**：检查评分标准和规则配置，调整参数后重新质检

### 5.3 报告问题

**问题**：报告下载失败
**解决方案**：检查报告路径是否正确，确保有足够的磁盘空间

**问题**：报告内容不完整
**解决方案**：确保质检过程正常完成，未被中断

## 6. 系统维护

### 6.1 日志管理

- 系统日志存储在`logs`目录下
- 按日期生成日志文件：`质检日志_YYYY-MM-DD.log`
- 日志包含系统运行状态、错误信息、质检结果等

### 6.2 缓存管理

- 缓存文件存储在`.cache`目录下
- 缓存包含上次质检结果，系统重启后会自动恢复
- 可通过"清除缓存"按钮手动清除缓存

### 6.3 数据库管理

- 系统使用本地数据库存储配置和历史数据
- 定期备份数据库文件，防止数据丢失

## 7. 最佳实践

### 7.1 数据准备

- 确保数据文件格式正确，包含必要的列
- 预处理数据，去除无关信息
- 对于大型文件，建议拆分为多个小文件处理

### 7.2 规则配置

- 根据业务需求调整评分标准
- 定期更新规则，适应业务变化
- 结合规则引擎和LLM优势，提高质检准确性

### 7.3 结果分析

- 定期分析质检结果，识别常见问题
- 针对高频错误制定改进措施
- 跟踪历史趋势，评估改进效果

### 7.4 成本控制

- 合理设置LLM调用参数，控制Token使用
- 优先使用规则引擎，减少LLM调用
- 定期监控费用，优化质检策略
